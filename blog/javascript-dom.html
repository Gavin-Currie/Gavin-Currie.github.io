JavaScript and the DOM


JavaScript
JavaScript: The engine behind HTML. The scripting language which give your pages more functionality.
Usually seen in the < script > section of your page, or as a link to a seperate file in it's own folder, like this:
< script src="/js/main.js" >
Now, this Javascript can hold various information about your elements within the web page.
Oh, another thing: Javascript has a reputation for being finnicky with text, so make sure your functions or variables are written in camel case; like this: firstSecondThirdword

Under the hood:

To have a better idea of how Javascript can manage your content, we need to look into the inspection console.
This is a portal to the various manangement controls and navigation of the structure of your pages.

To do this, on your web page your veiwing, right click on the page, then click: inspect.


This will bring up an inspection panel to the right of your browser.


When you are testing web pages with Javascript, including the line: console.(log) as a test output, is a good habit to adopt.

Components:
Javascript contains various components with which to operate and manage your data used within your content, such as:

  • Comments:Notes used to explain what blocks of code do, it's unseen by the program, and dosent affect the actual code. Written as: // your message...
  • Variables:The basic building blocks of Javascript code. Written as: let welcome = 'Hello world'
  • Functions:Reusable blocks of code that run the program to improve efficiency. Written as: function addTwoNumbers(x, y) { return x + y; }
  • Conditionals:For A/B, true/false testing. Written as: if ("cat" === "dog") { var outcome = "if block"; } else { var outcome = "else block"; } outcome;
  • Arrays:Used for handling multiple values. Written as: var breakfast = ["coffee", "croissant"];
  • Objects:Adding an extra dimension to values by giving them properties. Written as key/value pairs. Like this: const car = {type:"Fiat", model:"500", color:"white"};


The DOM
When they mention The DOM, it refers to the Document Object Model.
This how Javascript interacts with your web page.

Document: refers to Javascript handling the Web page as an HTML document.
Object: Javascript handles the HTML tags as objects and how the document heirarchy is structured like a 'tree'. (See my HTML page) With each nested element, Javascript treats as a child object or branches within that parent object.
Model: This is displayed as a representation of that JavaScript object. This includes:
  • The interfaces and objects used to represent and manipulate a document
  • The semantics of these interfaces and objects - including both behavior and attributes
  • The relationships and collaborations among these interfaces and objects



Interactivity
This where the DOM starts to show it's usefulness.

For example: you can have elements assigned with a call (references to) to a function residing in a JavaScript file.
Like this:< button onlick="addNum()">< button /> this points to the function within JavaScript.
example: function addNum() { run the function.. }

Showing results:
To take it up a notch, we can display the results of this interactivity as content on the page.
For example: say we have an H2 tag with an id of: addOne < h2 id="addOne"> < h2/>
Then, in the JavaScript, we add a variable that references that id.
like this: let addNum = document.getElementById("addOne")
Then in the function 'addNum', we include the variable from the H2 tag. like this: addNum.innerText = "two"
What this does, is the JavaScript targets that particular tag with that id and displays on the page (the .innerText part) within the H2 tag as: "two" (or whatever other variable you want displayed.)
