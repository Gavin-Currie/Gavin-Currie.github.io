The DOM
When they mention The DOM, it refers to the Document Object Model.

This how Javascript interacts with your web page.
Document: refers to Javascript handling the Web page as an HTML document.
Object: Javascript handles the HTML tags as objects and how the document heirarchy is structured like a 'tree'. (See my page on HTML). With each nested element, Javascript treats as a child object or branches within that parent object.
Model: This is displayed as a representation of that JavaScript object. This includes:

  • The interfaces and objects used to represent and manipulate a document
  • The semantics of these interfaces and objects - including both behavior and attributes
  • The relationships and collaborations among these interfaces and objects

Showing results:

To take it up a notch, we can display the results of this interactivity as content on the page.
For example - say we have:

  • A button with an id of "increment-btn"
  • A button with an id of "save-btn"
  • An H2 tag where the increment result is displayed
  • And the paragraph tag where the saved counts are displayed

Then in the Javascript, it all comes together.

What this does is, every time the increment button is clicked, the script tells the increment function to increase the count by one.

  • It assigns a count variable and sets it to zero
  • It then assigns a variable countEL. which references the HTML document, runs the method that targets the tag within the document by id (.getElementById) - yep, the H2 tag with the id of ("count-el") is what is being targeted.
  • Then in the increment function, it increases the count by one (count +=1), then spits out the result
    (the .innerText part is where javascript puts the code into HTML ) back into the H2 tag with the new variable count.
  • The save button does something similar: it assigns a variable saveEL which the method that targets the ("save-el") id, (the paragraph tag)
  • Then in the save function, with the variable countStr that saves the current total of the variable count, adds the " - ",
    Then inputs the (countStr) variable into .textContent (the paragraph tag)
  • It then resets the program, and the count variable back to zero, while still retaining the saved count total.
