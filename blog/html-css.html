HTML and CSS

HTML

HTML is the backbone to the Structure of Webpages. HTML stands for HyperText Markup Langauge.

All Webpages are handled as interconnected (linked) documents within a website as well as througout the entire internet.

It uses a particular scripting language that all browsers can process, format into a hierarchical order based on what elements are within the document, and displays it a way that users can understand.
Think of it like a russian doll (a wooden doll, inside a wooden doll, inside a wooden doll)



HTML encapsulates these elements (body, header, dividers, tables, lists, links, images, buttons, forms, paragraphs, footer...) within tags.
These tags use the greater than symbol, and the less than symbol.

All these tags must have an opening tags (< tag >) and closing tags (< /tag >). If any of these elements dont have either matching tags, HTML leaks out onto the page and displays gibberish.

CSS

Now, CSS stands for Cascading StyleSheets.
CSS are the rulesets that dictate the placement and formatting (styling) of said elements within the document, and HTML can have links within HTML documents referrring to CSS files
if you needed to upgrade the standard look of pages. Make it pop.


What does CSS look like? Here's a screen shot of some CSS in the wild.


These rules include attribrutes regarding the font and size of text, tables, images, video and links to other documents and its placement within the documents.
Attributes that are used include: padding, borders, colors, corners, buttons, position, display, margins, float, position, indent, z-index (layering), font-family, height, overflow, transforms... the list is pretty extensive.
Each of these attributes implements different characteristics of how the element looks in relation to elements next to it.
You can really fine tune how your content looks and behaves on the page by using various attributes in the rules for each element.
Fine tuning can include assigning classes and or ID's to particular elements to differentiate from other elements.
For example: .main points to < div class="main"> residing in the hmtl file, and #main points to < div id="main">

You can either have CSS in the same page within the < stlyle > section of the page or as a external link. Like this: < link rel="stylesheet" href="/styles/my.css">

CSS can also have nested elements and nested attributes with each ruleset (hence: the cascading part of it)


The Box Model


CSS also has a hierarchical order when it comes to the placement of elements. They use what is called: The Box model.
By default, all your elements (tables, buttons pages, images) come without styling, and look crammed next to each other on the page.
This where the box model comes into play.
Also, think of it like a russian doll.
Ok, so from the outside in, the order goes: Margin, Border, Padding, then the content (text or image, or whatever).



So, if you use this box model with skill and discernment, your content will look great on the page!!
